## 2.5：上线

- **预计用时：** 15 分钟

> **互动课程：** 在 Claude Code 里运行 `/start-2-5`

**本课目标：** 部署你的项目并拿到可分享的线上 URL。

## 什么是部署（Deployment）？

部署就是把代码从你电脑搬到互联网。

代码从你的机器 → 任何人都能访问的服务器 → 真实 URL。

开发者会不断说 “deploy / ship / go live”。这是你把东西变成“真的上线可用”的时刻。

## 什么是托管（Hosting）？

托管意味着：有服务器 24/7 运行你的代码，并处理访问流量。

你不需要自己运维服务器——托管平台负责一切。你只要把代码交给它。

## 什么是 Vercel？

Vercel 是一个网站托管服务：
- 个人项目**免费**（免费额度很友好）
- **由 Next.js 团队打造** - 配合非常顺畅
- **与 GitHub 连接** - 你 push 代码后自动部署

把 GitHub repo 交给 Vercel → 它构建你的代码 → 你得到一个真实 URL，比如 `yourproject.vercel.app`。

## 自动部署怎么工作？

这就是魔法：

1. 你把代码 push 到 GitHub
2. Vercel 检测到变更
3. Vercel 自动重新构建并发布
4. 你的线上网站在 ~1 分钟内更新

初次设置完成后基本没有手工步骤：改代码 → push → 就上线。

## 什么是 Vercel CLI？

Vercel CLI 是一个在终端里直接部署的工具。

Claude 会用它在不离开 Claude Code 的情况下完成部署。它同样通过浏览器认证，流程类似 GitHub。

## 常见问题排查

### “vercel: command not found”
没有安装 Vercel CLI。运行 `npm i -g vercel`

### Authentication failed
重新运行 `vercel login` 并完成浏览器授权。

### Build failed
去 Vercel 控制台查看错误日志。通常是代码问题——把错误信息告诉 Claude，它可以帮你修。

### “Project not found”
确认你在正确的项目目录里运行部署命令。

### 网站显示旧版本
可能仍在构建中。去 Vercel 控制台看状态——通常需要 1 分钟左右。

### 本地能跑，Vercel 不行
这是环境差异导致。查看 Vercel 构建日志，找出差异点。

## 部署之后

上线后：

1. **用手机打开测试** - 同一个 URL 任何设备都能访问
2. **分享给朋友** - 发链接，看他们的反应
3. **后续更新：** 改代码 → push 到 GitHub → 自动部署

这就是完整流程。你已经会把东西做出来并上线了。

## 完整闭环

你学会了：

| 步骤 | 含义 |
|------|---------------|
| **规划** | 把要做什么说清楚 |
| **构建** | 让 AI 做出来 |
| **迭代** | 不断修改到满意 |
| **保存** | 用 GitHub 备份 |
| **上线** | 部署到互联网 |

这套循环适用于任何项目：落地页、工具、应用、原型——同一流程。
