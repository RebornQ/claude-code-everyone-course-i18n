## 2.2：规划

- **预计用时：** 20-25 分钟

> **互动课程：** 在 Claude Code 里运行 `/start-2-2`

**本课目标：** Claude 会采访你，生成一份需求规格说明。

## 什么是需求（Requirements）？

需求是在构建之前，对“你要做什么”的描述。

你可以把它当成你和“负责实现的人”之间的契约：前期越具体，后期 surprises 越少。

无论你在和谁合作都一样：
- AI 工具
- 开发者
- 设计师
- 代理公司

模糊指令 = 意外结果。清晰需求 = 你真正想要的结果。

## 软件里的“逻辑”是什么？

逻辑是行为规则，也就是“如果这样，就那样”的那一层。

每个应用都有逻辑：
- 如果用户点击按钮 → 展示下一屏
- 如果选择答案 A → 给性格 X 加 1 分
- 如果某个分数最高 → 展示对应结果

你用自然语言定义逻辑，Claude 用代码实现它。

## 推荐系统是怎么工作的？

测验、推荐器、匹配系统都遵循同一模式：

1. **用户输入** → 答案、偏好、选择
2. **计分系统** → 每个输入为不同结果加权
3. **输出** → 分数最高的结果胜出

以性格测试为例：
- 每个答案映射到某个性格类型
- 答案不断累积分数
- 最终结果 = 分数最高的性格类型

你不用写代码——只需要决定映射关系。

## 为什么“视觉预览”很重要？

文字很容易有歧义。“现代感”对不同人意味着不同东西。

看一眼比描述更快：你通常“看到了就知道是不是对的”。

Claude 可以生成多套 HTML 预览文件，并在浏览器中打开。你选择喜欢的版本——或者把多个版本里喜欢的元素拼在一起。

## REQUIREMENTS.md 文件

在本课结束时，你的所有决定都会被写进一个文件：`REQUIREMENTS.md`

它包含：
- 你要构建什么
- 逻辑（如何运作）
- 视觉风格偏好
- 内容（题目、结果等）

下一课 Claude 会以这个文件作为规格说明进行构建。
