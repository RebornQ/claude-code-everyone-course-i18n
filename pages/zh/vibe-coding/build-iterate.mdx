## 2.3：构建与迭代

- **预计用时：** 30-40 分钟

> **互动课程：** 在 Claude Code 里运行 `/start-2-3`

**本课目标：** Claude 会搭脚手架、构建应用，然后你用截图做反馈不断迭代。

## 什么是开发服务器（Development Server）？

开发服务器会在本地运行你的代码，让你像访问网站一样看到效果。

你需要知道：
- 它只在**你的电脑**上可用（不是公开的）
- 它让你在上线之前反复测试与修改
- 当你在浏览器里查看时，它必须持续运行
- 在终端里按 `Ctrl+C` 可以停止

## 什么是 Localhost？

`localhost` 意味着：你的电脑在充当服务器。

`localhost:3000` = 你的电脑上 3000 端口。

这不是一个真实 URL，别人无法访问。你会在 2.5 部署后拿到真实 URL。

## 什么是 Plan 模式？

Plan 模式是 Claude Code 的一种输入模式：在执行前先展示计划。

- 用 **Shift+Tab** 切换
- 适合大型任务，先审计划再执行
- 你确认计划后，Claude 才开始写代码

对复杂构建来说，这能让你在落笔前先把问题问清楚、把方向校准好。

## 为什么迭代很重要？

第一版永远不完美——这很正常。

专业开发永远是：构建 → 评审 → 调整 → 重复。

AI 的区别是：迭代从“几天”变成“几秒”。所以别将就——一直改到你喜欢为止。

## 截图技巧

**Ctrl+V**（Mac 不是 Cmd+V！）可以把图片直接粘贴进 Claude Code。

这对迭代非常强：
- 截图你觉得不对的地方
- 圈出问题或指给 Claude 看
- 粘贴并描述你想怎么改

视觉反馈比文字描述更快、更准确。

## 常见问题排查

### “npm: command not found”
没有安装 Node.js。去 [nodejs.org](https://nodejs.org) 下载并安装，然后重启终端。

### “Port 3000 already in use”
另一个应用正在占用端口。你可以：
- 关闭可能正在运行服务器的其他终端窗口
- 改用其他端口（让 Claude 帮你即可）

### 浏览器没有自动打开
手动访问 [http://localhost:3000](http://localhost:3000)

### 页面空白
1. 看终端是否有构建错误（红色文本）
2. 打开浏览器控制台：按 F12，切到 “Console”，看红色错误

### 改动没有生效
- 刷新浏览器了吗？
- 开发服务器还在终端里运行吗？

### 样式看起来不对
清理缓存：**Cmd+Shift+R**（Mac）或 **Ctrl+Shift+R**（Windows），或试试无痕窗口。
